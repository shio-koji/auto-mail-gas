# Gmail 差し込みメール送信システム 要件定義

## 背景・目的

Google Spreadsheet + GAS を使い、宛先ごとに内容を差し込んで Gmail から一斉送信する仕組みを作る。
非エンジニアでも使いやすい UI を重視する。

---

## 決定事項まとめ

| 項目 | 決定内容 |
|------|----------|
| メールテンプレートの場所 | スプレッドシート内の専用シート（`テンプレート`） |
| メール形式 | まずプレーンテキスト。必要があれば、HTML対応を追加 |
| 送信トリガー | カスタムメニューのボタンを押して手動実行 |
| 宛先の管理 | 「送信対象」チェックボックス列でON/OFF選択する |
| 差し込み変数 | 列ヘッダー名を自動検出（自由に列を追加できる） |
| プレースホルダー記法 | `{{列ヘッダー名}}` （例: `{{名前}}`） |
| CC / BCC | 不要（宛先のみ） |
| 差出人名 | Google アカウントの表示名をそのまま使用（変更不要） |
| Gmail 署名 | GAS 送信では自動付与されないため、テンプレートシートに署名欄を用意して本文末尾に結合する |
| 送信ステータス列の役割 | **ログ専用**。送信の可否判断には使わない。送信対象チェックのみで送る相手を決める |
| メインの操作フロー | 下書き作成（Gmail で確認してから手動送信）。本番送信（直接送信）は副次的 |

---

## スコープ外の機能
- プレーンテキストではなくて、HTMLのリッチテキストのメールを送る（可能）
- 時刻指定でメールを送信する（可能）
- CC/BCCの機能（可能）
- スプレッドシートでメールテンプレートを書くのではなく、gmail側で下書きを作成してから、スプレッドシート側でテンプレを選択する。（可能）
- 時間指定送信（可能）

## スプレッドシート構造

### シート1: `宛先リスト`

| 列 | 内容 | 備考 |
|----|------|------|
| A | 送信対象 | チェックボックス（TRUE/FALSE） |
| B | メールアドレス | 必須 |
| C | 名前 | 差し込みフィールド例 |
| D | 所属名 | 差し込みフィールド例 |
| ... | （任意追加可） | ヘッダー名が `{{}}` 記法に対応 |
| 最後から2列目 | 送信ステータス | `下書き作成済み` / `送信済み` / `エラー: ...` （**ログ専用**。送信可否の判断には使わない） |
| 最終列 | 操作日時 | 下書き作成または送信時に自動記録 |

- ステータス列・操作日時列はヘッダー名（`送信ステータス`, `操作日時`）で検出する
- **送信対象チェック（A列）だけが「誰に送るか」を決める。ステータス列は参照しない**

### シート2: `テンプレート`

A列にラベル、B列に値を入力する形式。

| A列（ラベル） | B列（値） | 備考 |
|------|------|------|
| 件名 | 〇〇のご案内 | 差し込み記法使用可 |
| 本文 | （本文テキスト） | 差し込み記法使用可、複数行対応 |
| 署名 | （署名テキスト） | 空欄可。本文末尾に `--` 区切りで自動結合 |

- 差出人名は Google アカウントの表示名をそのまま使用する。

**送信時のメール本文の組み立て:**

```
{{本文の内容}}

--
{{署名の内容}}
```

- 署名が空欄の場合は区切り線ごと省略する
- 署名には差し込み記法は使用しない（固定テキスト）

## 機能一覧

### カスタムメニュー（`onOpen` で追加）

```
メール送信ツール
├── テスト送信（自分に1件送る）          ← 差し込みの確認用
├── 下書き作成（Gmail の下書きに保存する）← メイン操作
├── 本番送信（送信対象に全員に直接送信する）← 副次的・要注意
└── ログをクリア（ステータス列・操作日時列を空欄に戻す）
```

### テスト送信

- 宛先リストの最初にチェックが入っている行のデータを使う
- チェックが1件もない場合は「少なくとも1件チェックを入れてください」とエラーを出す
- 件名に `[差し込みメール送信テスト]` プレフィックスを付ける
- 送信先は自分（スクリプト実行者）の Gmail アドレス
- 差し込みが正しく機能しているかを確認する用途

### 下書き作成（メイン操作）

1. 「送信対象」が TRUE の行を全件対象に抽出（**ステータスは参照しない**）
2. 対象件数を表示して確認ダイアログ: 「○件の下書きを作成します。よろしいですか？」
3. 1件ずつ差し込み処理 → `GmailApp.createDraft()` で下書き作成 → ステータス・操作日時を記録
4. エラー時も止まらず次の行へ進む（エラー内容をステータス列に記録）
5. 完了後にトースト通知:「○件の下書きを作成しました。×件のエラーがありました。Gmail の下書きフォルダを確認して送信してください。」

**利用者の操作フロー:**
1. スプレッドシートで送信対象にチェックを入れる
2. 「下書き作成」を実行
3. Gmail の下書きフォルダを開いて内容を確認
4. 問題なければ Gmail から手動で送信

### 本番送信（副次的・危険操作）

1. 「送信対象」が TRUE の行を全件対象に抽出（**ステータスは参照しない**）
2. 確認ダイアログ 1回目: 「○件のメールを直接送信します。よろしいですか？」
3. 確認ダイアログ 2回目: 「本当に○件に送信します。この操作は取り消せませんが、送信しますか？」
4. 1件ずつ差し込み・送信・ステータス更新を繰り返す
5. エラー時も止まらず次の行へ進む（エラー内容をステータス列に記録）
6. 完了後にトースト通知:「○件送信完了、×件エラー」

### 差し込みロジック

**テンプレート駆動方式**を採用する。列のハードコード除外リストは持たない。

1. テンプレートの件名・本文から `{{...}}` を正規表現でスキャンし、使われている変数名一覧を抽出する
2. 宛先リストのヘッダー行（1行目）を読み込み、変数名と一致する列のインデックスを取得する
3. 宛先行ごとに、変数名 → 対応するセルの値 に置換する

```
例:
  テンプレート本文: "{{名前}} 様、{{所属名}} の〇〇です。"
  → 変数: ["名前", "所属名"]
  → 宛先リストのC列(名前)・D列(所属名) の値で置換
```

- テンプレートに `{{...}}` が書かれていない列は一切参照しない
- 対応する列が宛先リストに存在しない場合は空文字に置換し、エラーにはしない（ログに警告を残す）

## GAS ファイル構成

```
auto-mail-gas/
├── appsscript.json   # oauthScopes 設定
├── config.gs         # 定数（シート名・ヘッダー名・プレースホルダー記法）
├── ui.gs             # onOpen・カスタムメニュー・ダイアログ・トースト
├── sheet.gs          # スプレッドシートの読み書き（宛先リスト取得・ステータス更新）
├── template.gs       # テンプレートシート読み込み・差し込み置換処理
└── mail.gs           # GmailApp の薄いラッパー（下書き作成・送信）
```

| ファイル | 責務 | 将来の拡張例 |
|---------|------|------------|
| config.gs | 定数の一元管理 | シート名変更時ここだけ修正 |
| ui.gs | メニュー・ダイアログのみ | サイドバーUI追加 |
| sheet.gs | スプレッドシート操作のみ | 複数シート対応 |
| template.gs | 差し込み処理のみ | HTML テンプレート対応 |
| mail.gs | Gmail 操作のみ | 添付ファイル対応・CC追加 |

> **GAS の注意**: ファイルを分けても全て同一グローバルスコープ。関数名の衝突を避けるため、各ファイルの関数名はファイル名を接頭辞にする（例: `sheet_getRecipients()`, `template_merge()`）。

### `appsscript.json` に追加するスコープ

```json
"oauthScopes": [
  "https://www.googleapis.com/auth/spreadsheets",
  "https://www.googleapis.com/auth/gmail.send",
  "https://www.googleapis.com/auth/gmail.compose",
  "https://www.googleapis.com/auth/script.container.ui"
]
```

## 送信制限・クォータ

| API | 制限 | 備考 |
|-----|------|------|
| `GmailApp.sendEmail()` | 100通/日（無料）、1,500通/日（Workspace） | 送信先アドレスの数でカウント |
| `GmailApp.createDraft()` | 20,000回/日 | 現実的に上限に達しない |
| GAS 実行時間 | 6分/実行 | 超過するとスクリプトが強制終了 |

**`Utilities.sleep()` は不要**: `GmailApp` には per-minute のレート制限がない。`Utilities.sleep()` は「短時間呼びすぎ」エラーが実際に出た場合の対処法であり、予防的に入れる必要はない。

**残りクォータの確認（本番送信前に使うと安全）:**
```javascript
const remaining = MailApp.getRemainingDailyQuota();
// → 今日あと何通送れるかを返す
```

## 検証方法（動作確認手順）

1. スプレッドシートを開き、カスタムメニュー「メール送信ツール」が表示されることを確認
2. `宛先リスト` シートに自分の Gmail アドレスを 3行入力、チェックをON
3. `テンプレート` シートに件名・本文・署名（`{{名前}}` など差し込みあり）を記入
4. **テスト送信** を実行 → 自分宛に 1件届き、差し込みと署名が正しく機能しているか確認
5. **下書き作成** を実行 → 確認ダイアログが出て、Gmail の下書きフォルダに 3件作成されることを確認
6. `送信ステータス` 列が「下書き作成済み」になり、`操作日時` が記録されていることを確認
7. **ログをクリア** を実行 → ステータス列・操作日時列が空欄に戻ることを確認
8. 再度 **下書き作成** を実行 → チェックがONの行に再び下書きが作成されることを確認（ステータスに関係なく動くことの確認）
9. 1行のメールアドレスを意図的に壊してエラーを起こし、ステータス列に `エラー: ...` が記録されることを確認
10. **本番送信** を実行 → 二重確認ダイアログが 2回出ることを確認し、3件が実際に届くことを確認
