# Gmail 差し込みメール送信システム 実装設計書

> **位置づけ**: 要件定義（requirments.md）をもとに、「どのファイルに・どの関数を・どういう責務で実装するか」を記述する。コーディング前の設計図として使う。

---

## ファイル一覧と責務

```
auto-mail-gas/
├── appsscript.json   # GAS の設定ファイル（スコープ定義）
├── config.gs         # 定数の一元管理
├── ui.gs             # メニュー・ダイアログ・トーストなど UI 制御
├── sheet.gs          # スプレッドシートの読み書き
├── template.gs       # テンプレート読み込み・差し込み置換
└── mail.gs           # Gmail 操作（下書き作成・送信）
```

> **GAS の注意**: 全ファイルは同一グローバルスコープ。関数名の先頭にファイル名を付けて衝突を防ぐ。

---

## `appsscript.json`

コードではなく設定ファイル。以下のスコープを `oauthScopes` に追加する。

```json
{
  "timeZone": "Asia/Tokyo",
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8",
  "oauthScopes": [
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/gmail.send",
    "https://www.googleapis.com/auth/gmail.compose",
    "https://www.googleapis.com/auth/script.container.ui"
  ]
}
```

## `config.gs` — 定数の一元管理

関数は持たない。定数（`const`）のみを定義する。

```
定数一覧:

SHEET_RECIPIENTS       = '宛先リスト'       // 宛先リストシートの名前
SHEET_TEMPLATE         = 'テンプレート'      // テンプレートシートの名前

HEADER_TARGET          = '送信対象'          // チェックボックス列のヘッダー名
HEADER_EMAIL           = 'メールアドレス'    // メールアドレス列のヘッダー名
HEADER_STATUS          = '送信ステータス'    // ステータス列のヘッダー名
HEADER_TIMESTAMP       = '操作日時'          // 操作日時列のヘッダー名

TEMPLATE_LABEL_SUBJECT   = '件名'
TEMPLATE_LABEL_BODY      = '本文'
TEMPLATE_LABEL_SIGNATURE = '署名'

PLACEHOLDER_REGEX      = /\{\{(.+?)\}\}/g   // {{変数名}} を検出する正規表現

STATUS_DRAFT           = '下書き作成済み'
STATUS_SENT            = '送信済み'
STATUS_ERROR_PREFIX    = 'エラー: '          // エラー時は 'エラー: <内容>' の形式
```

## `sheet.gs` — スプレッドシートの読み書き

### `sheet_getHeaderMap()`

```
目的    : 宛先リストの1行目（ヘッダー行）を読み込み、ヘッダー名 → 列インデックス のマップを返す
引数    : なし
戻り値  : Object { 'メールアドレス': 1, '名前': 2, '所属名': 3, ... }  (0始まり)
処理    :
  1. 宛先リストシートを取得
  2. 1行目を getRange(1, 1, 1, lastColumn).getValues()[0] で取得
  3. 各セル値をキー、インデックスを値にしたオブジェクトを返す
```

### `sheet_getTargetRows()`

```
目的    : 「送信対象」が TRUE の行のデータを全件取得する
引数    : なし
戻り値  : Array of Object
          [
            { rowIndex: 2, email: 'a@example.com', data: ['田中', '営業部', ...] },
            ...
          ]
          rowIndex は実際の行番号（1始まり, ヘッダーは1行目なので最小2）
処理    :
  1. sheet_getHeaderMap() でヘッダーマップを取得
  2. 2行目以降の全データを getValues() で取得
  3. HEADER_TARGET 列が TRUE の行だけフィルタリング
  4. 各行を { rowIndex, email, data } の形に整形して返す
```

### `sheet_getFirstTargetRow()`

```
目的    : テスト送信用。送信対象のうち最初の1行だけ返す
引数    : なし
戻り値  : Object { rowIndex, email, data } または null（チェックがない場合）
処理    : sheet_getTargetRows() の結果の[0]番目を返す
```

### `sheet_updateStatus(rowIndex, status)`

```
目的    : 指定行のステータス列と操作日時列を更新する
引数    :
  rowIndex  : 行番号（1始まり）
  status    : 書き込む文字列（例: '下書き作成済み', 'エラー: ...'）
戻り値  : なし
処理    :
  1. sheet_getHeaderMap() でステータス列・操作日時列の列番号を取得
  2. ステータス列に status を書き込む
  3. 操作日時列に new Date() を書き込む
```

### `sheet_clearLog()`

```
目的    : 全行のステータス列・操作日時列を空欄に戻す（ログをクリア）
引数    : なし
戻り値  : なし
処理    :
  1. 全データ行を取得
  2. ステータス列・操作日時列の範囲を clearContent() で空欄にする
```

### `sheet_setup()`

```
目的    : 初回セットアップ。必要なシート・ヘッダー・書式を自動で作成する
引数    : なし
戻り値  : なし
処理    :
  【宛先リストシートの作成】
  1. SHEET_RECIPIENTS 名のシートが存在するか確認
     - 存在しない場合: insertSheet() で新規作成
     - 存在する場合: そのまま使う（上書きしない）
  2. A1〜F1 にヘッダーを書き込む
     ['送信対象', 'メールアドレス', '名前', '所属名', '送信ステータス', '操作日時']
  3. A列（2行目以降）にチェックボックスを設定
     setDataValidation(SpreadsheetApp.newDataValidation().requireCheckbox().build())
     ※ 2行目〜1000行目など十分な範囲に適用
  4. 1行目をフリーズ（freezeRows(1)）
  5. ヘッダー行に背景色を設定（視認性向上）
  6. 列幅を調整（autoResizeColumns）

  【テンプレートシートの作成】
  7. SHEET_TEMPLATE 名のシートが存在するか確認
     - 存在しない場合: insertSheet() で新規作成
     - 存在する場合: そのまま使う（上書きしない）
  8. A列にラベル、B列に説明プレースホルダーを書き込む
     A2: '件名'   B2: '（例: 〇〇のご案内）'
     A3: '本文'   B3: '（例: {{名前}} 様、...）'
     A4: '署名'   B4: '（例: 田中太郎 / 03-xxxx-xxxx）'
  9. A列を太字・背景色で装飾して「ラベル列」とわかるようにする
  10. B列の幅を広げる（テンプレート記入しやすくするため）

  【完了】
  11. ui_showToast('セットアップが完了しました。宛先リストとテンプレートシートを確認してください。')
```

## `template.gs` — テンプレート読み込み・差し込み置換

### `template_read()`

```
目的    : テンプレートシートから件名・本文・署名を読み込む
引数    : なし
戻り値  : Object { subject: '...', body: '...', signature: '...' }
処理    :
  1. テンプレートシートの A 列（ラベル列）を全行取得
  2. TEMPLATE_LABEL_SUBJECT / BODY / SIGNATURE のラベルがある行の B 列を取得
  3. 3つの値をオブジェクトにして返す
  4. ラベルが見つからない場合はエラーをスロー
```

### `template_extractVariables(subject, body)`

```
目的    : 件名・本文中の {{変数名}} をスキャンして変数名の配列を返す
引数    :
  subject : 件名の文字列
  body    : 本文の文字列
戻り値  : Array of String（重複なし）例: ['名前', '所属名']
処理    :
  1. subject + body に PLACEHOLDER_REGEX を適用
  2. マッチした全キャプチャグループ（変数名）を Set に入れて重複を除く
  3. Array に変換して返す
```

### `template_merge(subject, body, headerMap, rowData)`

```
目的    : 1行分のデータを差し込んで、件名と本文の置換済み文字列を返す
引数    :
  subject   : 件名テンプレート文字列
  body      : 本文テンプレート文字列
  headerMap : sheet_getHeaderMap() の戻り値
  rowData   : 1行分のセルデータ配列（getValues()[i] の形式）
戻り値  : Object { subject: '置換済み件名', body: '置換済み本文' }
処理    :
  1. template_extractVariables() で変数名一覧を取得
  2. 各変数名について headerMap から列インデックスを取得
  3. 列インデックスが存在する場合: rowData[index] で値を取得
     存在しない場合: '' （空文字）に置換し、console.warn でログ出力
  4. subject と body の {{変数名}} を replace() で置換して返す
```

### `template_buildBody(body, signature)`

```
目的    : 本文と署名を結合してメールの最終的な本文を組み立てる
引数    :
  body      : 置換済みの本文
  signature : 署名テキスト（空欄の場合は '' or null）
戻り値  : String
処理    :
  - 署名が空欄: body をそのまま返す
  - 署名がある: body + '\n\n--\n' + signature を返す
```

## `mail.gs` — Gmail 操作

### `mail_getMyEmail()`

```
目的    : スクリプト実行者（自分）の Gmail アドレスを取得する
引数    : なし
戻り値  : String（メールアドレス）
処理    : Session.getActiveUser().getEmail() を返す
```

### `mail_getRemainingQuota()`

```
目的    : 今日あと何通送信できるかを返す
引数    : なし
戻り値  : Number
処理    : MailApp.getRemainingDailyQuota() を返す
```

### `mail_createDraft(to, subject, body)`

```
目的    : Gmail の下書きを1件作成する
引数    :
  to      : 宛先メールアドレス
  subject : 件名
  body    : 本文
戻り値  : なし
処理    : GmailApp.createDraft(to, subject, body) を呼ぶ
```

### `mail_send(to, subject, body)`

```
目的    : メールを1件送信する
引数    :
  to      : 宛先メールアドレス
  subject : 件名
  body    : 本文
戻り値  : なし
処理    : GmailApp.sendEmail(to, subject, body) を呼ぶ
```

## `ui.gs` — メニュー・ダイアログ・トースト

### `onOpen()`

```
目的    : スプレッドシートを開いた時にカスタムメニューを追加する
引数    : なし
戻り値  : なし
処理    :
  SpreadsheetApp.getUi().createMenu('メール送信ツール')
    .addItem('テスト送信（自分に1件送る）', 'ui_menuTestSend')
    .addItem('下書き作成（Gmail の下書きに保存する）', 'ui_menuCreateDrafts')
    .addSeparator()
    .addItem('本番送信（全員に直接送信する）', 'ui_menuSendAll')
    .addSeparator()
    .addItem('ログをクリア', 'ui_menuClearLog')
    .addSeparator()
    .addItem('初期セットアップ（シートを自動作成する）', 'ui_menuSetup')
    .addToUi()
```

### `ui_menuTestSend()`

```
目的    : 「テスト送信」メニューの実行処理
引数    : なし
戻り値  : なし
処理    :
  1. sheet_getFirstTargetRow() で最初のチェック行を取得
  2. null の場合: ui_showAlert('少なくとも1件チェックを入れてください') して終了
  3. template_read() でテンプレート取得
  4. sheet_getHeaderMap() でヘッダーマップ取得
  5. template_merge() で差し込み
  6. template_buildBody() で本文組み立て
  7. mail_getMyEmail() で自分のアドレス取得
  8. mail_send(自分のアドレス, '[差し込みメール送信テスト] ' + 件名, 本文)
  9. ui_showToast('テスト送信完了')
```

### `ui_menuCreateDrafts()`

```
目的    : 「下書き作成」メニューの実行処理
引数    : なし
戻り値  : なし
処理    :
  1. sheet_getTargetRows() で送信対象行を取得
  2. 0件なら ui_showAlert('送信対象が0件です。チェックを入れてください。') して終了
  3. ui_showConfirm('○件の下書きを作成します。よろしいですか？')
     → キャンセルなら終了
  4. template_read() / sheet_getHeaderMap() を取得
  5. 各行をループ:
     a. template_merge() で差し込み
     b. template_buildBody() で本文組み立て
     c. try { mail_createDraft() } catch(e) { sheet_updateStatus(rowIndex, STATUS_ERROR_PREFIX + e.message); continue }
     d. sheet_updateStatus(rowIndex, STATUS_DRAFT)
  6. ui_showToast('○件の下書きを作成しました。Gmail の下書きフォルダを確認して送信してください。')
```

### `ui_menuSendAll()`

```
目的    : 「本番送信」メニューの実行処理
引数    : なし
戻り値  : なし
処理    :
  1. sheet_getTargetRows() で送信対象行を取得
  2. 0件なら alert して終了
  3. mail_getRemainingQuota() を確認
     → 残り < 対象件数 の場合: 'クォータ不足の警告' を出して続行確認
  4. 確認ダイアログ1回目: '○件のメールを直接送信します。よろしいですか？'
     → キャンセルなら終了
  5. 確認ダイアログ2回目: '本当に○件に送信します。この操作は取り消せません。'
     → キャンセルなら終了
  6. 各行をループ:
     a. template_merge() → template_buildBody()
     b. try { mail_send() } catch(e) { sheet_updateStatus(rowIndex, STATUS_ERROR_PREFIX + e.message); continue }
     c. sheet_updateStatus(rowIndex, STATUS_SENT)
  7. ui_showToast('○件送信完了、×件エラー')
```

### `ui_menuClearLog()`

```
目的    : 「ログをクリア」メニューの実行処理
引数    : なし
戻り値  : なし
処理    :
  1. ui_showConfirm('ステータスと操作日時の記録をすべて消去します。よろしいですか？')
     → キャンセルなら終了
  2. sheet_clearLog()
  3. ui_showToast('ログをクリアしました')
```

### `ui_menuSetup()`

```
目的    : 「初期セットアップ」メニューの実行処理
引数    : なし
戻り値  : なし
処理    :
  1. ui_showConfirm('シートの初期セットアップを行います。既存のシートは上書きされません。よろしいですか？')
     → キャンセルなら終了
  2. sheet_setup()
  ※ sheet_setup() 内で完了トーストを出すため、ここでは追加の通知不要
```

### `ui_showConfirm(message)`

```
目的    : Yes/No の確認ダイアログを表示し、Yes なら true を返す
引数    : message（表示するメッセージ）
戻り値  : Boolean
処理    :
  const result = SpreadsheetApp.getUi().alert(message, Ui.ButtonSet.YES_NO)
  return result === Ui.Button.YES
```

### `ui_showAlert(message)`

```
目的    : OK ボタンのみのアラートを表示
引数    : message
戻り値  : なし
処理    : SpreadsheetApp.getUi().alert(message)
```

### `ui_showToast(message)`

```
目的    : 右下にトースト通知を表示
引数    : message
戻り値  : なし
処理    : SpreadsheetApp.getActiveSpreadsheet().toast(message, 'メール送信ツール', 5)
```

## 関数の呼び出し関係図

```
onOpen()
  └── メニュー登録のみ

ui_menuTestSend()
  ├── sheet_getFirstTargetRow()
  │     └── sheet_getHeaderMap()
  ├── template_read()
  ├── template_merge()
  │     └── template_extractVariables()
  ├── template_buildBody()
  ├── mail_getMyEmail()
  └── mail_send()

ui_menuCreateDrafts()
  ├── sheet_getTargetRows()
  │     └── sheet_getHeaderMap()
  ├── template_read()
  ├── sheet_getHeaderMap()
  ├── [ループ]
  │     ├── template_merge()
  │     ├── template_buildBody()
  │     ├── mail_createDraft()
  │     └── sheet_updateStatus()
  └── ui_showToast()

ui_menuSendAll()
  ├── sheet_getTargetRows()
  ├── mail_getRemainingQuota()
  ├── [ループ] （ui_menuCreateDrafts と同構造、mail_send を使用）
  └── ui_showToast()

ui_menuClearLog()
  └── sheet_clearLog()

ui_menuSetup()
  └── sheet_setup()
        └── ui_showToast()（完了通知）
```

## 実装の順序（推奨）

1. `config.gs` — 定数の定義（他の全ファイルが依存する）
2. `sheet.gs` — スプレッドシート操作（テスト可能な基礎部分）
3. `template.gs` — 差し込みロジック（sheet.gs に依存）
4. `mail.gs` — Gmail 操作（単独でテスト可能）
5. `ui.gs` — メニューと各処理の組み立て（全ファイルに依存）
6. `appsscript.json` — スコープ追加（デプロイ前に確認）
